# Testing Tracker

Track tested pages for each role. Check the box when page testing is completed.
Only roles that have access to a page have checkboxes.

---

## Legend
- ✅ = Tested & Passed
- [ ] = Not Yet Tested
- ❌ = Role does not have this page
- ⚠️ = Tested with Issues (note in comments)

---

## Test Categories per Page
Each page should be tested for:
1. **UI** — Layout, styling, responsiveness
2. **Navigation** — Links, routing, breadcrumbs
3. **Data** — Correct data loading, display, filtering
4. **Actions** — Buttons, forms, CRUD operations
5. **Permissions** — Role-based access enforced

## How to Use

1. Test each page by logging in with the corresponding role
2. Replace `[ ]` with `✅` when a page passes all test categories
3. Replace `[ ]` with `⚠️` if issues are found — log them in the Bug Log
4. Add notes in the "Notes" column for quick reference
5. Update the Testing Summary counts as you progress

---

## Test Checklist (per page)

- [ ] Page loads without errors
- [ ] Layout renders correctly on desktop
- [ ] Layout renders correctly on mobile/tablet
- [ ] Navigation links work correctly
- [ ] Data loads and displays properly
- [ ] Filters and search work as expected
- [ ] CRUD operations function correctly (if applicable)
- [ ] Form validations are enforced
- [ ] Error states handled gracefully
- [ ] Unauthorized roles are blocked from access
- [ ] No console errors or warnings
